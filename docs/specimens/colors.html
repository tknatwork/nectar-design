Foundations · Color

Colors

Each named color is a ramp, not a swatch. Eleven steps from 50 through 500 (the seed) to 950, so every hue carries its own legibility, surface, and emphasis variants. The page itself paints from the engine — drag the scrubber to see the chrome bend.

Tint
Hue mixed toward white. Surfaces, hover backgrounds, large fills.
Tone
Hue mixed toward grey — saturation drops, lightness barely moves. Muted text, secondary surfaces.
Shade
Hue mixed toward black. Primary text, active states, deep emphasis.

Heat × Depth engine

runtime · oklch

The portfolio's chrome doesn't pick from the ramps directly — it computes oklch(L C var(--dynamic-hue)) per element. Heat shifts hue, Depth shifts lightness. The ramps are the authoring vocabulary; the engine is what paints.

Heat axis
--ui-heat 0 → 100 · hue 250° → 40°
Depth axis
--ui-depth 0 → 100 · L 0.96 → 0.15
Authoring rule. Static surfaces (slides, prints, throwaway prototypes) pick from the ramps by step number — e.g. honey-100 for a card surface, honey-700 for an active CTA. Live portfolio surfaces consume the engine variables: oklch(var(--L-bg) var(--C-bg) var(--dynamic-hue)) or the bridged aliases (--bg, --fg, --primary, --surface, --muted-fg, --border).
