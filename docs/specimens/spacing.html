Spacing, radii, glass, motion

Everything is on a 4px grid. Radii read from the GlassCard scale (16–36px). Shadow is one cast — --glass-shadow — varied by depth via backdrop blur. Motion runs short and snappy by default; spring stiffness/damping are the only "long" values in the system.

Glass — one shadow, three depths

There is no second surface treatment. Where a card needs to feel heavier, raise the depth — only backdrop-filter changes; tint comes from the engine.

depth=surface

blur(12px) saturate(2)
inline tiles, small chips

depth=raised

blur(16px) saturate(2)
project cards, content blocks

depth=floating

blur(24px) saturate(2)
hero, nav pill, modals

Motion — hover the cards

Default is 200ms with --ease-out. Anything longer feels slow on a 60Hz display. prefers-reduced-motion drops every transition to 0ms.

--dur-instant · 50ms · ease-out-circ
--dur-fast · 120ms · ease-in-out
--dur-base · 200ms · ease-out (default)
--dur-slow · 300ms · ease-out-back

Springs (Framer Motion · verbatim from GlassCard.tsx)

Two springs ship. Cards use the soft one; buttons use the snappy one.

--spring-card
stiffness 300, damping 20

GlassCard hover lift. Soft settle, ~280ms.

--spring-button
stiffness 500, damping 25

Action button scale-to-1.04. Snappier; ~180ms.

Cross-fade — circadian engine

When solar position drives a variable change, the transition is intentionally long: 60 seconds. It's ambient, not interactive.

--dur-cross · 60s
color, background-color, border-color, fill, stroke
