# Spacing

A 4px base unit scale with semantic aliases for consistent rhythm across
every component and layout. Migrated from
`myportfolio/app/app/design-system/spacing/page.tsx` (Wave J, May 2026).

## Semantic Aliases

Alias tokens give spacing values purpose. Use these instead of raw numbers.

| Alias | Maps to | Use for |
| --- | --- | --- |
| `xs` | 4px | Tight internal gaps |
| `sm` | 8px | Small component gaps |
| `md` | 16px | Default spacing |
| `lg` | 24px | Section padding |
| `xl` | 32px | Large gaps |
| `2xl` | 48px | Major separations |
| `section` | 96px | Between page sections |
| `touch` | 44px | **WCAG 2.5.5 minimum tap target** (Wave H, `--touch-target`) |

The `touch` alias encodes intent: 44px is the WCAG 2.5.5 minimum
non-negotiable accessibility dimension. Use `min-h-touch min-w-touch`
on tappable elements (nav links, buttons, form controls).

## Radius scale

One tight px-valued scale — **`map.borderRadius`** — is the single source. It
feeds Tailwind's `rounded-*` utilities (via `@theme --radius-*`) and *every*
surface (UI controls + atmospheric/glass). There is no parallel "soft" scale.

| Token | Utility | Value |
| --- | --- | --- |
| `--radius-none` | `rounded-none` | 0 |
| `--radius-sm` | `rounded-sm` | 4px |
| `--radius-md` | `rounded-md` | 8px |
| `--radius-lg` | `rounded-lg` | 12px |
| `--radius-xl` | `rounded-xl` | 16px |
| `--radius-2xl` | `rounded-2xl` | 24px |
| `--radius-full` | `rounded-full` | 9999px |

Content surfaces consume the same scale via `--card-radiusSm/Md/Lg`
(8 / 12 / 16px); glass panels use `--radius-2xl` (24px). These were a separate
10/14/20 + 24/36 atmospheric scale before the radius consolidation — now
collapsed onto the one scale.

## Breakpoints

Mobile-first. Stack at `mobile`, side-by-side at `tablet`, optimized at
`desktop`, full bleed at `wide`.

| Name | Value | Device |
| --- | --- | --- |
| `mobile:` | 390px | iPhone 15/16 baseline |
| `tablet:` | 768px | iPad portrait |
| `desktop:` | 1024px | Laptop |
| `wide:` | 1440px | QHD+ full bleed |

Use `mobile:`, `tablet:`, `desktop:`, `wide:` — never the default
Tailwind `sm:`, `md:`, `lg:`. The cva `size` props (`sm`, `md`, `lg`) are
component variants, not responsive prefixes.

## Layout grid

| Surface | Columns | Gutter | Margin |
| --- | --- | --- | --- |
| Mobile | 4 columns | 16px | 16–20px |
| Tablet | 8 columns | 24px | 32px |
| Desktop | 12 columns | 24–32px | 40–48px |

Reading width caps at 65ch for prose, 80ch for code.

## z-index named layer

| Token | Value | Use |
| --- | --- | --- |
| `--z-modal` | 9999 | AccessGate / GateOverlay (kept above all other UI) |

Accessed via Tailwind utility `z-modal`. The only named z-index in the
system; everything else uses Tailwind's standard scale (`z-0`, `z-10`,
`z-50`). Wave H promoted `z-[9999]` from a magic number to a named layer.
