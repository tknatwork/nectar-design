# Colors

A warm, pastel-forward palette built from seed primitives through semantic
aliases to theme-mapped values. Migrated from `myportfolio/app/app/design-system/colors/page.tsx` (Wave J, May 2026) — this Storybook is the source of truth.

They change between light and dark mode (and breathe with the engine when
the value is engine-driven).

## Toast colors

Toast notification colors are scoped to `GlassToast` only (not bridged to
Tailwind utilities). Distinct from the static brand status colors above —
toasts want attention-grabbing brightness for ephemeral notifications.

```css
--toast-default: oklch(var(--L-accent) 0.12 var(--dynamic-hue));
--toast-success: oklch(0.72 0.16 145);
--toast-error:   oklch(0.65 0.20 25);
--toast-warning: oklch(0.75 0.16 80);
```

## Pastel Seeds

Raw primitive colors — the building blocks. Named for what they look like,
not what they mean. Live as `--seed-color-*` in the Style Dictionary
pipeline (Tier 2). Examples: `lavender`, `mint`, `peach`, `coral`, `sage`,
`blush`, `sky`, `lemon`, `butter`, `honey`, `cream`, `fog`, `ash`, `cloud`,
`wisteria`, `azure`, `spring`, `lichen`.

## Neutral Scale

11 steps from `50` (near-white `#FAFAF9`) to `950` (near-black `#0C0A09`).
Tailwind's neutral palette is preserved 1:1 — components reach for these
when they need a flat backdrop unrelated to Heat × Depth.

## How Colors Flow

Seed defines the raw value (e.g. `honey = #FFE082`). Alias gives it
meaning (`primary → honey`). Mapped resolves per theme (light primary =
honey, dark primary = deep honey `#D4A843`). Components reference mapped
tokens via CSS custom properties — they never see raw hex.

The runtime **Heat × Depth engine** then overrides the mapped tier for
every paint-relevant token at `:root` in `app/globals.css`. See
`Foundations/Theming` for the full engine docs.
