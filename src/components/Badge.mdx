## Anatomy

A badge is a single non-interactive `<span>`: a short text label inside a tinted
pill. Geometry (height, padding, radius) is locked per `size`; the tint — a 10%
wash of the intent colour over matching foreground text — is set per `intent`.

## Usage

- **Do** — pick the `intent` that matches the meaning (`success` for healthy, `warning` for attention, `destructive` for failure) so colour reinforces the label.
- **Don't** — use a badge as a button; it has no focus ring or hit target. For an interactive pill use `Tag` (removable) or `Button`.

## Accessibility

- A badge is decorative text — the label itself carries the meaning, so screen readers read it as ordinary content.
- Never signal state by colour alone; the word (`Active`, `Failed`) must say it too, for colour-blind and monochrome contexts.
- The 10% intent wash keeps text/background contrast high — keep custom overrides above 4.5:1.
- Use the **Accessibility** tab on any story canvas for the live axe audit.
