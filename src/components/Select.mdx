## Anatomy

A trigger button (styled like `Input` — border, focus ring, error state) showing the
current value or `placeholder` and a chevron, plus a portalled popup of `options`.
Each item shows its label and a check on the selected row; per-option `disabled` is
supported. Trigger geometry is locked per `size`.

## Usage

- **Do** — keep `options` to a scannable list for one-of-many choice; mark unavailable entries with per-option `disabled`.
- **Don't** — reach for `Select` when only two or three mutually exclusive choices exist; a `RadioGroup` shows them all at once.

## Accessibility

- Built on Base UI's accessible select: full keyboard support (arrows, type-ahead, Enter, Escape) and managed focus.
- Pair with a `<Label htmlFor>` matching the trigger's `id`, or pass `aria-label`; the `error` state can be wired via `aria-describedby` + `aria-required`.
- The popup renders in a portal but stays in the accessibility tree as a listbox owned by the trigger.
- Use the **Accessibility** tab on any story canvas for the live axe audit.
