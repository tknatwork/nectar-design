## Anatomy

A heading is a single line (or wrapped run) of display-font text. The `level`
prop drives both the rendered tag (`<h1>`–`<h6>`) and the scale — size, weight,
and line-height step together down the ramp. Use `as` to keep the visual scale
while overriding the tag, so the document outline stays correct.

## Usage

- **Do** — pick `level` for the document outline first, then reach for `as` if the visual size needs to differ from the semantic rank.
- **Don't** — jump straight to `level={1}` for emphasis on a sub-section, or skip a level (h2 → h4) to get a smaller size — use `as` instead.

## Accessibility

- `level` controls the rendered `<h1>`–`<h6>` — visual size is **not** semantic level; an `as`-overridden heading keeps its scale but changes its tag.
- Don't skip heading levels: a screen-reader outline reads ranks in order, so an h2 followed by an h4 reads as a gap.
- One `<h1>` per page (the page title); `Heading` defaults to `level={2}` precisely so sections don't accidentally claim the top rank.
- Use the **Accessibility** tab on any story canvas for the live axe audit.
