## Anatomy

A `RadioGroup` renders one labelled circular control per `options` entry. The selected
ring borders in `primary` and fills with a `primary` dot; the group lays out vertically
by default or horizontally via `orientation`. Per-option `disabled` dims a single choice;
group-level `disabled` dims them all.

## Usage

- **Do** — use a radio group when exactly one of several visible options must be chosen, and pre-select a sensible `defaultValue`.
- **Don't** — use it for independent toggles (use `Checkbox`) or for long option lists that would scroll (use `Select`).

## Accessibility

- Built on Base UI's `RadioGroup`: arrow keys move selection within the group, which is a single Tab stop.
- Name the group with `aria-label` (or an associated heading); each control inherits its option `label` as its accessible name.
- `disabled` (per-option or group-wide) is reflected to assistive tech, not just dimmed visually.
- Use the **Accessibility** tab on any story canvas for the live axe audit.
