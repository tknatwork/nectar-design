## Anatomy

A tag is a bordered, fully-rounded `<span>` holding a text label and — when an
`onRemove` handler is passed — a trailing icon `<button>`. Border, fill, and text
colour come from `variant`; padding and text size from `size`. The radius is
always `rounded-full`.

## Usage

- **Do** — use tags for many-of-a-set metadata (filters, topics, skills); add `onRemove` when the user can dismiss them.
- **Don't** — use a tag as the primary action of a view, or omit `onRemove` but style it to look dismissible.

## Accessibility

- The remove control is a real `<button type="button">` with `aria-label="Remove"` — it is keyboard-focusable and Enter/Space activates it.
- It carries its own `focus-visible:ring-ring`, so keyboard users see exactly which tag they're about to remove.
- Drive removal from `onRemove`, not colour or position — the label still has to read on its own.
- Use the **Accessibility** tab on any story canvas for the live axe audit.
