## Anatomy

A button is one interactive element: an optional leading icon, a text label, and a
focus ring. Geometry (height, padding, radius) is locked per `size`; colour per
`intent`. It always renders a native `<button>`.

## Usage

- **Do** — one `primary` button per view for the main action; pair `destructive` with a confirm step.
- **Don't** — stack competing `primary` buttons, or rely on colour alone for `destructive` (the label should say so too).

## Accessibility

- Native `<button type="button">` — no accidental form submit.
- Focus-visible ring via `--ring` (2px offset).
- `disabled` leaves the tab order; **also gate the handler** — dimming alone isn't a disabled state.
- Use the **Accessibility** tab on any story canvas for the live axe audit.
