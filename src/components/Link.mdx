## Anatomy

A link is a native `<a>`: a text label, an optional trailing external-link
glyph, and a focus ring. The `variant` sets colour and underline treatment;
`external` adds `target="_blank"`, the safe `rel`, and the arrow icon. It always
renders inline.

## Usage

- **Do** — use `default` for inline prose links, `muted` for secondary references, and `nav` (no underline) inside navigation bars; set `external` for off-site destinations.
- **Don't** — drop the underline on inline body links (`nav` is for nav bars only), or write bare "click here" labels that don't describe the destination.

## Accessibility

- Discernible text — the label should make sense out of context; avoid "here"/"this link".
- Focus-visible ring via `--ring` with a 2px offset, so keyboard focus is always visible across all variants.
- `external` opens a new tab and sets `rel="noopener noreferrer"`; the trailing arrow is `aria-hidden`, so name the destination in the text (e.g. "… (opens in a new tab)") when the new-window behaviour matters.
- Use the **Accessibility** tab on any story canvas for the live axe audit.
