## Anatomy

`List` is a flex column that renders a `<ul>` (or `<ol>` when `ordered`). Its
`variant` sets row rhythm: `default` flush, `spaced` with a small gap, or `divided`
with separator lines. Each `ListItem` is an `<li>` with its own `size` (row padding)
and an `interactive` flag that adds a pointer cursor and hover wash.

## Variants

Two independent axes split across the container and the item: `List` owns
`variant` (`default` / `spaced` / `divided`), while `ListItem` owns `size`
(`sm` / `md` / `lg`) and `interactive`. The **Variants**, **Sizes**, and
**Interactive** stories demonstrate each.

## Usage

- **Do** — use `variant="divided"` for dense settings rows, and mark rows `interactive` only when clicking them actually does something.
- **Don't** — set `ordered` for content where sequence is irrelevant; the `<ol>` numbering implies an order to assistive tech.

## Accessibility

- Renders a real `<ul>`/`<ol>` of `<li>` items, so screen readers announce list semantics and item counts.
- Choose `ordered` only when sequence carries meaning — it switches the element to `<ol>` and exposes position.
- An `interactive` row is styled but not itself a control; nest a `<button>`/`<a>` (or wire keyboard handlers) so it's reachable and operable by keyboard.
- Use the **Accessibility** tab on any story canvas for the live axe audit.
