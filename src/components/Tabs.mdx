## Anatomy

Tabs is a compound built on Base UI: a `TabsList` holds the `TabsTrigger` buttons, and each
`TabsContent` panel maps to a trigger by `value`. The selected trigger lifts onto a `surface`
chip; exactly one panel is mounted-visible at a time. Selection is `defaultValue` (uncontrolled)
or `value` + `onValueChange` (controlled).

## Usage

- **Do** — use for 2–5 peer views of the *same* subject (Overview / Activity / Settings); keep labels to one or two words.
- **Don't** — use tabs for a linear sequence (that's a wizard) or to hide content a screen reader user needs at once.

## Accessibility

- Base UI wires the WAI-ARIA roles: `role="tablist"` on the list, `role="tab"` on each trigger, `role="tabpanel"` on each panel, linked via `aria-controls`/`aria-labelledby`.
- Roving tabindex — arrow keys move between triggers, the active tab is the single tab stop, and `disabled` triggers are skipped by keyboard and click.
- Focus-visible ring via `--ring` on the active trigger.
- Use the **Accessibility** tab on any story canvas for the live axe audit.
