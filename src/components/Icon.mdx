## Anatomy

An icon is a single inline glyph rendered by Iconify from an `icon` identifier
(`set:name`, e.g. `ph:house-duotone` or `lucide:search`). Box size is locked per
`size`; colour is inherited from the surrounding text via `currentColor`, so any
`text-*` utility recolours it.

## Usage

- **Do** — colour icons with text utilities (`text-primary`, `text-muted-fg`) so they track the theme; pick a consistent icon set per surface.
- **Don't** — convey state by icon shape alone without a label; pair it with text or an `aria-label` when it carries meaning.

## Accessibility

- Decorative by default — with no `aria-label` the icon is `aria-hidden`, so screen readers skip it (correct when adjacent text already conveys the meaning).
- Pass `aria-label` to make it semantic: that switches it to `role="img"` with an accessible name, for standalone icon-only controls.
- Inheriting `currentColor` keeps icon/background contrast in lockstep with the text it sits beside.
- Use the **Accessibility** tab on any story canvas for the live axe audit.

## Tokens used

Icon defines no colour tokens of its own — it inherits `currentColor`, so it picks up whatever semantic text colour (`--fg`, `--primary`, `--muted-fg`, …) the surrounding `text-*` utility resolves to. Size maps to fixed Tailwind box utilities (`size-4` … `size-8`).
