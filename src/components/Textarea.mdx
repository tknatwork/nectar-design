## Anatomy

A native `<textarea>` that mirrors `Input`'s border, focus ring, and error treatment,
but stacks vertical padding for multiple lines and allows vertical resize (`resize-y`).
Set `rows` for the initial height; it forwards its ref for form integration.

## Usage

- **Do** — set a sensible `rows` default for the expected content length, then let the user resize vertically.
- **Don't** — disable resize or cap height so tightly that long entries scroll inside a two-line box.

## Accessibility

- The `error` variant sets `aria-invalid="true"`; wire the message in with `aria-describedby` (or use `FormField`).
- Give it a real label via `<Label htmlFor>` or `aria-label` — placeholder text disappears on input and isn't announced as a name.
- `disabled` drops it from the tab order and excludes its value from submission.
- Use the **Accessibility** tab on any story canvas for the live axe audit.
